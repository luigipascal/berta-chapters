## How to Contribute Code/Content

### For Small Changes (typos, minor fixes)

1. **On the chapter's GitHub page**, click the pencil icon (✏️) on the file you want to edit
2. **Make your changes** directly in the browser
3. **Add a description** of your change
4. **Click "Propose changes"**
5. **Create a pull request**
6. We'll review and merge! ✨

### For Larger Changes (new sections, exercises, code)

1. **Fork the repository**
   - Click "Fork" on the chapter's GitHub page
   - This creates your own copy

2. **Clone your fork**
   ```bash
   git clone https://github.com/YOUR-USERNAME/berta-chapter-XX-topic.git
   cd berta-chapter-XX-topic
   ```

3. **Create a branch**
   ```bash
   git checkout -b your-feature-name
   ```
   - Examples: `fix-typo-ch3`, `add-advanced-exercise`, `improve-diagram`

4. **Make your changes**
   - Edit files
   - Add new notebooks/scripts
   - Update documentation
   - Test your changes work!

5. **Commit your changes**
   ```bash
   git add .
   git commit -m "Clear description of what you changed"
   ```

6. **Push to your fork**
   ```bash
   git push origin your-feature-name
   ```

7. **Create a pull request**
   - Go to the original repository
   - Click "New pull request"
   - Select your branch
   - Add a title and description
   - Click "Create pull request"

8. **Respond to feedback**
   - We may ask for changes
   - Make updates on your branch
   - They automatically appear in the PR
   - We merge when ready!

## Contribution Guidelines

### Quality Standards

**Code:**
- Must work (tested on Python 3.10+)
- Include docstrings and comments
- Follow PEP 8 style guide
- Avoid external dependencies if possible

**Notebooks:**
- Clear explanations with code
- Good-quality output/visualizations
- Progressive difficulty
- Executable without errors
- Well-organized cells with markdown

**Documentation:**
- Clear and concise
- Proper markdown formatting
- Spellchecked
- Grammatically correct
- Links work and are appropriate

**Exercises:**
- Solvable by chapter-level learners
- Include hints if needed
- Have complete solutions
- Test multiple concepts
- Provide learning value

### What We Look For

✅ **We love:**
- Thoughtful, well-executed improvements
- Contributors who test their changes
- Clear commit messages and PR descriptions
- Responses to feedback and review comments
- Respect for the curriculum's philosophy (practical, accessible, rigorous)

❌ **We avoid:**
- Untested code
- Marketing/spam
- Off-topic changes
- Poor formatting
- Disrespectful tone

### Attribution

All contributors are credited! When we merge your PR:

- Your name appears in the commit history
- Major contributions get mentioned in the chapter README
- You can add yourself to a CONTRIBUTORS.md file
- Your GitHub profile will be linked
