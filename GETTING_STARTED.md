## Get Started (5 Minutes)

### Step 1: Clone the Repository

```bash
git clone https://github.com/luigipascal/berta-chapters.git
cd berta-chapters
pip install -r requirements.txt
```

### Step 2: Launch the Interactive Hub

```bash
python interactive/berta.py
```

The hub offers:
- **Skill Assessment** to find where to start
- **Learning Paths** for guided progression
- **Progress Tracking** across chapters
- **Knowledge Quizzes** to test yourself

### Step 3: Pick a Chapter

Choose from available chapters:
- **Chapter 1**: [Python Fundamentals](./chapters/chapter-01-python-fundamentals/)
- **Chapter 2**: [Data Structures & Algorithms](./chapters/chapter-02-data-structures/)
- **Chapter 3**: [Linear Algebra & Calculus](./chapters/chapter-03-linear-algebra/)

Or browse the full curriculum in [SYLLABUS.md](./SYLLABUS.md).

### Step 4: Open the First Notebook

```bash
cd chapters/chapter-01-python-fundamentals
jupyter notebook notebooks/01_introduction.ipynb
```

### Step 5: Learn by Doing

Follow along with:
- 📖 Explanations and theory
- 💻 Code examples you run
- 🧪 Exercises to solve
- 📚 Production scripts to study

That's it! You're learning.

## How Each Chapter is Structured

Every Berta chapter repository contains:

```
berta-chapter-XX-topic/
├── README.md                          # Start here
├── notebooks/
│   ├── 01_introduction.ipynb         # Beginner (start here)
│   ├── 02_intermediate.ipynb         # Intermediate
│   └── 03_advanced.ipynb             # Advanced (optional)
├── scripts/
│   ├── main_application.py           # Production code
│   └── utilities.py                  # Helper functions
├── exercises/
│   ├── problem_set_1.ipynb           # Do these!
│   ├── problem_set_2.ipynb
│   └── solutions/
│       └── (solutions in solutions branch)
├── assets/
│   └── diagrams/                     # SVG and mermaid diagrams
├── datasets/
│   └── (sample data and links)
└── requirements.txt                  # Install with pip
```

**Recommended flow:**
1. Read the README
2. Run `01_introduction.ipynb` 
3. Pause and do exercises
4. Run `02_intermediate.ipynb` if interested
5. Try writing your own code using `scripts/`

## Completing a Chapter

Here's how to get the most from each chapter:

### 📖 Read First
- Start with the README (learning objectives, prerequisites, time estimate)
- Skim the first notebook to understand what you'll learn

### 💻 Code Along
- Open `01_introduction.ipynb`
- **Don't just watch—code along!**
- Read explanations
- Run the code
- Modify examples
- Understand *why*, not just *what*

### 🧪 Do the Exercises
- Open the exercises notebooks
- Solve problems without looking at solutions first
- Struggle a little—that's learning!
- Check solutions in the `solutions/` branch when stuck
- Try to improve the solutions

### 🏗️ Build Something
- Use `scripts/main_application.py` as inspiration
- Build your own variation
- Apply to a problem you care about
- Share what you built!

### 🤝 Contribute Back
- Found an error? Open an issue
- Have improvements? Submit a PR
- Want to add examples? Fork and contribute

**Time estimate:** 
- Quick chapter (6 hours) = 1-2 weeks at 5-7 hours/week
- Deep chapter (12 hours) = 2-3 weeks at 5-7 hours/week

## Moving to the Next Chapter

You've completed a chapter? Great!

### Check Prerequisites
Before starting chapter N+1:
- Open its README
- Check "Prerequisites"
- Make sure you've done the earlier chapters

### No Skipping (Usually)
Chapters build on each other. Don't skip ahead unless:
- ✅ You're very experienced (jump to Advanced Track)
- ✅ You're skipping optional topics
- ✅ A README explicitly says it's independent

### When to Move On
Move to the next chapter when:
- ✅ You understand the core concepts
- ✅ You've completed the exercises
- ✅ You can explain ideas to someone else
- ✅ You've applied concepts to something real

Don't worry about:
- ❌ Being 100% perfect (you won't be)
- ❌ Remembering every detail (you'll look things up)
- ❌ Moving "fast enough" (everyone learns at different paces)

## Common Questions

### Q: Do I need to do all chapters?
**A:** No! Do what's relevant to your goals. Check the [learning paths](./CURRICULUM.md#learning-paths) in CURRICULUM.md.

### Q: Can I skip chapters?
**A:** Depends on the chapters. Check prerequisites in each chapter's README. Foundation chapters build heavily on each other; Practitioner chapters are more modular.

### Q: How long does this take?
**A:** Highly variable. Each chapter estimates time. Foundation: 38 hours, Practitioner: 88 hours, Advanced: 84 hours. You decide your pace.

### Q: What if a chapter is too hard?
**A:** 
1. Re-read the prerequisites
2. Do the exercises slowly
3. Ask for help in the GitHub issues
4. Request a custom "prerequisite chapter" if needed
5. Try different explanations online (chapters complement other resources)

### Q: What if a chapter is too easy?
**A:** 
1. Jump to the advanced notebook
2. Modify the exercises to make them harder
3. Apply concepts to your own project
4. Request a more advanced chapter on the topic

### Q: Can I use these chapters professionally/commercially?
**A:** Yes! All chapters are under the MIT License. You can use them to build products, teach courses, etc. Just attribute Berta AI and Luigi Pascal Rondanini.

### Q: I'm completely lost. Where do I start?
**A:** Take a breath! Learning is hard. Here's your path:
1. Read this page (✓ you're doing it!)
2. Read [CURRICULUM.md](./CURRICULUM.md) — just skim it
3. Look at [learning paths](./CURRICULUM.md#learning-paths) — find one that matches you
4. Clone the first chapter in your path
5. Follow the steps in "Get Your First Chapter" above

You've got this! 💪
